# SheetSync — ব্যবহার গাইড (প্লাগইন অ্যাক্টিভ থেকে প্রোডাক্ট আপলোড)

**কার জন্য:** যারা Google Sheet দিয়ে WooCommerce-এ প্রোডাক্ট আপলোড/আপডেট করবেন।

**এই গাইডে:**

1. প্লাগইন **অ্যাক্টিভ করার পর** প্রথমে কী করবেন  
2. Sheet থেকে **প্রোডাক্ট আপলোড** কীভাবে করবেন  
3. **Simple** ও **Variable** প্রোডাক্টের পার্থক্য  

বিস্তারিত কলাম তালিকা: **[PRODUCT-UPLOAD-GUIDE.md](PRODUCT-UPLOAD-GUIDE.md)** (ইংরেজি)  
ধাপে ধাপে ইংরেজি সংস্করণ: **[GETTING-STARTED.md](GETTING-STARTED.md)**  
**সিঙ্ক আটকে গেলে / ক্রন সেটআপ:** **[ব্যাকগ্রাউন্ড-সিঙ্ক-ও-ক্রন-গাইড.md](ব্যাকগ্রাউন্ড-সিঙ্ক-ও-ক্রন-গাইড.md)**  
**ভেরিয়েবল প্রোডাক্ট:** **[VARIABLE-PRODUCTS-GUIDE.md](VARIABLE-PRODUCTS-GUIDE.md)** (ইংরেজি)

## অংশ ১ — প্লাগইন অ্যাক্টিভ করার পর (একবার করুন)

### ধাপ ১: প্রয়োজনীয় জিনিস

| জিনিস | আছে কি? |
|-------|---------|
| WordPress ৬.০+ | ☐ |
| WooCommerce চালু | ☐ |
| একটি Google Sheet | ☐ |

### ধাপ ২: Google Service Account

1. [Google Cloud Console](https://console.cloud.google.com/) খুলুন  
2. প্রজেক্টে **Google Sheets API** চালু করুন  
3. **Service Account** বানান → **JSON key** ডাউনলোড করুন  
4. JSON-এর ভিতরে **ইমেইল** কপি করুন (যেমন: `xxx@yyy.iam.gserviceaccount.com`)

### ধাপ ৩: Sheet শেয়ার করুন

1. আপনার Google Sheet খুলুন  
2. **Share** → সেই **service account ইমেইল** দিন  
3. Permission: **Editor**  
4. Save  

**এটা না করলে প্লাগইন sheet পড়তে পারবে না।**

### ধাপ ৪: WordPress-এ JSON দিন

1. **SheetSync → Settings**  
2. Service Account **JSON** পুরোটা পেস্ট করুন  
3. **Save**

### ধাপ ৫: Products Connection বানান

1. **SheetSync → Connections** → নতুন connection  
2. গুরুত্বপূর্ণ ফিল্ড:

| ফিল্ড | কী দেবেন |
|-------|----------|
| Connection type | **Products** (Orders নয়) |
| Spreadsheet ID | Sheet URL-এর মাঝের অংশ |
| Sheet tab name | যেমন `Sheet1` |
| Header row | সাধারণত `1` (প্রথম row হেডার হলে) |

3. Save করুন  

### ধাপ ৬: Sheet-এ হেডার (Row ১)

প্রথম row-এ কলামের নাম। যেমন:

- SKU (Product Key)  
- Product Title  
- Regular Price  
- Stock Quantity  
- Product Type  
- Parent SKU (variable products)  
- Variation Attributes  
- ইত্যাদি  

**সহজ উপায়:** **SheetSync → Import / Export** → connection সিলেক্ট → **View Import Headers** ক্লিক করলে হেডার sheet-এ চলে আসতে পারে।

এখন প্রোডাক্ট row ভরা ও import করা যাবে।

## অংশ ২ — আপলোড কীভাবে কাজ করে (সহজ চিত্র)

```text
আপনি Google Sheet-এ row ভরেন
        ↓
SheetSync প্রতিটি row পড়ে
        ↓
কলাম → WooCommerce ফিল্ড (SKU, দাম, স্টক…)
        ↓
WooCommerce-এ প্রোডাক্ট তৈরি/আপডেট
```

| প্রোডাক্ট টাইপ | Sheet-এ কত row |
|----------------|----------------|
| **Simple** | ১ row = ১ প্রোডাক্ট |
| **Variable** | ১ parent row + প্রতিটি variation-এ ১ row |

**Order** সম্পর্কিত কলাম (Order ID, Customer Name…) **প্রোডাক্ট sheet-এ ভরবেন না** — খালি রাখুন।

## অংশ ৩ — Simple প্রোডাক্ট আপলোড

### Sheet-এ ১ row উদাহরণ

| SKU | Product Title | Regular Price | Stock | Status | Product Type | Parent SKU | Variation Attributes |
|-----|---------------|---------------|-------|--------|--------------|------------|----------------------|
| TSH-001 | Blue T-Shirt | 599 | 50 | publish | simple | খালি | খালি |

### WordPress-এ

1. **SheetSync → Import / Export**  
2. Connection সিলেক্ট → **View Import Headers**  
3. **Step 2:** প্রতিটি sheet কলাম → মিলিয়ে WooCommerce field বেছে নিন  
4. **SKU = Key field** চেক করুন  
5. **Create new products** চালু → **Start Import**  
6. **Products → All Products**-এ দেখুন

## অংশ ৪ — Variable প্রোডাক্ট আপলোড

### আগে WooCommerce-এ (একবার)

**Products → Attributes**

| Attribute | Terms (ছোট হাতের slug) |
|-----------|-------------------------|
| Color | `red`, `blue` |
| Size | `s`, `m` |

Sheet-এ লিখবেন: `pa_color`, `pa_size`

### Sheet-এ row (parent আগে, তারপর variation)

**Parent row (উদাহরণ)**

| SKU | Title | Type | Variation Attributes |
|-----|-------|------|----------------------|
| HOODIE-01 | Cotton Hoodie | variable | `pa_color:red,blue\|pa_size:s,m` |

- **Parent SKU** = খালি  
- দাম/স্টক parent-এ সাধারণত খালি (variation-এ দেবেন)

**Variation row ১**

| SKU | Parent SKU | Variation Attributes | Price | Stock |
|-----|------------|----------------------|-------|-------|
| HOODIE-01-RED-S | HOODIE-01 | `pa_color:red\|pa_size:s` | 899 | 10 |

**Variation row ২**

| SKU | Parent SKU | Variation Attributes | Price | Stock |
|-----|------------|----------------------|-------|-------|
| HOODIE-01-RED-M | HOODIE-01 | `pa_color:red\|pa_size:m` | 899 | 8 |

- Variation row-এ **Product Type** খালি  
- `|` = attribute আলাদা, `:` = নাম ও মান আলাদা  

### Import

আবার **Import / Export** → map (**Product Type**, **Parent SKU**, **Variation Attributes** অবশ্যই) → **Start Import**

WooCommerce-এ parent খুলে **Variations** ট্যাবে ২টা variation দেখা উচিত।

## অংশ ৫ — আপনি vs প্লাগইন

| আপনি করবেন | প্লাগইন করবে |
|------------|--------------|
| Sheet-এ ডেটা লিখবেন | Sheet পড়বে |
| কলাম ম্যাপ করবেন | WooCommerce-এ সেভ করবে |
| Variable-এর attribute বানাবেন | Parent + variation তৈরি/আপডেট |
| Sheet service account-এ share | Google API দিয়ে কানেক্ট |

---

## অংশ ৬ — প্রতিদিনের কাজ

**একবার অনেক প্রোডাক্ট:** Sheet ভরুন → **Import / Export** → **Start Import**

**নিয়মিত sheet থেকে আপডেট:** Sheet এডিট → **Connections** → **Sync Now**

## অংশ ৭ — ভুল যেন না হয়

| ভুল | ফলাফল |
|-----|--------|
| Sheet share করা হয়নি | Import ফেল |
| Connection = Orders | প্রোডাক্ট ঠিকমতো যাবে না |
| Variation parent-এর আগে | Variation skip |
| Variation row-এ `simple` | ভুল টাইপ |
| Order কলাম ভরা | কাজ করে না, শীট নষ্ট হয় |
| `Red` লিখা (`red` নয়) | Attribute মিল নাও করতে পারে |
